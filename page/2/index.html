c# 接口和抽象类 知识小记

发表于 2022-06-02 更新于 2022-08-06 分类于 编程
本文字数： 3.3k 阅读时长 ≈ 3 分钟

之所以把接口和抽象类放在一起说，是因为它们的含义相近但又有不同。相同的地方是，我们需要利用接口和抽象类来定义有什么，而让做什么放在更细分的类里。

  • 在抽象类中，我们可以创造需要由派生类实现的功能。

  • 在接口中，我们定义功能，但无法实现。派生类可以可以拓展接口并实现函数。

听起来感觉差不多，但是我们明确一下，抽象类是对类本质的抽象，表达的是 is 的关系，比如：SUV is a Car。接口是对行为的抽象，表达的是 like 的关系，比如：SUV like a Aircraft(假设SUV会飞），但是本质上还是 SUV is a Car.接口的核心是定义行为，对于实现主题是谁、是如何实现的，接口并不关心。

接下来看一下具体讲解和代码示范，一看便知道大概是怎么用的了，但是具体使用还是根据项目经验来

接口

接口有一些小细节和规则

  • 方法名和返回类型完全匹配
  • 所有参数（包括ref和out关键字修饰符）都完全匹配
  • 用于实现接口的所有隐式实现方法都必须具有public可访问性、但使用显示接口实现则不应该添加访问修饰符
  • 不能在接口中定义任何字段。字段本质上是类或结构的实现细节
  • 不能定义任何构造器和析构器，即构造函数和析构函数
  • 不能为任何方法指定访问修饰符。接口所有方法都隐式为公共方法
  • 不能再接口中嵌套任何类型（例如枚举、结构、类或其他接口)
  • 接口可以继承另一个接口，但不允许从结构或类继承。

抽象类

抽象类同样也有一些小细节和规则

  • 抽象类只作声明，而不包含实现，抽象方法旨在继承后实现，所以不能用private修饰
  • 抽象类不能被直接实例化
  • 抽象类可以没有抽象方法，但是有了抽象方法就一定是要声明类为抽象类
  • 抽象类的派生类不是抽象类，必须重写抽象类的所有抽象方法

最近在系统学习c#的知识点，有一些知识点比较散且使用频率不高，现在此记录，便于复习。

关于c#的异常处理

一句话概括，异常是程序执行期间出现的问题，异常处理的目的是为了维护程序的健壮性和稳定性。举个栗子，比如说除以0，用户输入的数据与预设数据类型不符，数组越界，栈溢出等等。c#给我们提供了一些易于使用的解决方法。主要由四个关键词组成（当然还有其他）： try，catch，throw，finally

  • try：用于存放可能会出现问题的语句，后面可以跟多个catch
  • catch：用于捕捉异常，通常跟在try后面
  • throw：用于抛出异常，是的，你没看错，比如说为了限制输入范围，限制输入的数字为1~10之间，主要用于自定义的限制
  • finally：不管是否有异常，里面的代码块必定执行，通常跟try一起。

c#中的异常类

C# 异常是使用类来表示的。C# 中的异常类主要是直接或间接地派生于 System.Exception 类。

System.ApplicationException 和 System.SystemException 类是派生于 System.Exception 类的异常类。

System.ApplicationException 类支持由应用程序生成的异常。所以程序员定义的异常都应派生自该类。

Blender 甜甜圈学习笔记（Blender Guru）

发表于 2022-05-28 更新于 2022-08-06 分类于 设计
本文字数： 2.3k 阅读时长 ≈ 2 分钟

许久未学习Blender，已经忘得差不多了，为了作业不得不捡回来，这次学习的是Youtube上的Blender Guru的Donuts（Blender版本3.0），算是很多人推荐的教程，可以学到很多。至此边学习便记录以便复习。

Part 1：Beginner Tutorial

第一部分算是对Blender进行一个基本认知，界面UI的了解，根据80/20原理，我们只需要认真学20%的知识，就能完成80%的任务，一开始对于复杂的UI界面会感到惊慌失措，无所适从，Just following，就能做出一个很好的3D模型，并且对于Blender有一个更深层次的认知。

对于3D建模软件来说，记住hotkey是十分重要的，一开始有太多的hotkey，当学到新的知识就会重复用到老的知识点，所以只要跟着教程做就可以记住，不必当心一下子就忘记了。

1.快速认识3d viewport，主要是装着3D模型的界面（除去右边的和上面的界面）

2.如果想看到渲染后的图片，可以在顶部导航栏点击Render，然后点击Render Image或者按下hotkey F12

3.移动物体，用鼠标左键选择物体，可以在3d viewport的左侧栏点击 移动键（长得像十字）这样会出现关于物体的XYZ坐标轴，值得注意的是这里的 Z 坐标是高度（与Unity不同，Unity Y是高度）。也可按下hotkey G来移动物体，按下G的同时按下鼠标中键可以跟着一根坐标轴移动，同样也可以G+X 、G+Y，G+Z来跟随坐标轴移动

4.旋转或者放大放小物体。同样在左侧栏可以找到，不过我们仍然习惯用hotkey，分别是R（Rotate），S（Scale），跟随坐标变化跟移动键相同

5.旋转viewport的视角，按住鼠标中键

6.对于鼠标按键的设置，不喜欢用中键的可以设置，laptop user without middle mouse button (Edit>Preferences>Input>Enable Emulate 3 Button Mouse, hold ALT + left click to orbiting)

7.根据喜好改变用户界面的大小（Edit>Preferences>Interface>Resolution Scale)

8.移动（panning）viewport的视角，用shift+中键

9.放大缩小viewport，用中键滚轮

10.快速调整视角有三种方法，1.用数字小键盘 2.按住 hotkey ~ ，然后选择视角 3.在显示ObjectMode的旁边有一个View按钮，View>ViewPoint>选择想要的视角（不推荐）4.用ALT+鼠标中键

11.右上的界面是这个文件里有多少东西，右下的是这个物体的属性，不过太多了，边用边学，现在不必了解。

12.删除物体，选择物体后有两个hotkey，一个是X，一个是Delete，凭借自己爱好使用。

13.复制物体，hotkey shift+D 拖到想要的位置鼠标左键为确定，鼠标右键为取消该次操作

Part 2：Object Editing Tutorial

这一Part主要是关于 Object 模式和 Editing 模式的介绍，没有介绍的那么详细，但是好用的功能基本是有了。

1.添加物体，hotkey shift +A ,或者是Viewport界面左上有一个Add按钮。

2.本教程做的是donut，我们选择Torus，然后会弹出一个初始化Torus选项的菜单 hotkey F9或者删掉重新建一个会出来菜单（一旦进行操作后不能修改）

3.快速弹出选择的物体的属性栏 hotkey N 或者直接点击右上方的 ← 箭头，可以直接拉回去，也可以用hotkey。

4.初始化属性参数，除了Dimensions，Location，Rotation以及Scale是可以初始化参数的，假设你变大了，用 hotkey Ctrl+A to apply > Scale ，可以使Scale重新变为（1，1，1）

5.在创建球体，环圈等带有弧线的物体，会出现锯齿线，只需要选择物体后 点击鼠标右键 弹出shade smooth/shade flat ，选择shade smooth，就可以变得较为光滑

6.即使shade smooth之后，边界也没有那么光滑，这时我们为其添加变形器,选择subdivision surface，他是可以增加表面的UV数，只看边界,两点构成一条直线，如果在其中间再加一个点，使其变得更像一条弧线。subdivision surface 就是做这个的，参数越高，越来越顺滑，不过基本3就够用了。

7.稍微讲解了Object模式的部分组件，接下来转到了编辑模式，编辑模式是只编辑你选择的物体。有几种方式可以快速调到Edit 模式（1.hotkey Tab 2.在左上方Object Mode的地方可以选择 3.Ctrl+Tab可以选择所有模式。）

8.开始调整上面的UV点，按住shift+左键可以多选，按G去移动点。

9.比例编辑，hotkey O可以转为比例编辑模式，比例编辑模式按G与普通模式按G的区别是：比例模式相当于牵一发而动全身，而普通模式只动选择的部分。开启比例编辑模式通常会出现一个圈来调整范围大小。

10.伸缩功能(shrink and flatten)在左侧可以选，hotkey Alt+S,即是让物体沿着正常路线移动

11.随机选择，左上角Select -> Select Random

12.在比例模式按钮旁边可以选择比例模式的类型（可以随便尝试一下）

关于这篇文章

学习本书也有一段时间，用博客来记录输出一些知识，巩固一下知识。《拾苹果》是本书的第一个数字化原型，也是最简单的，可以通过其初步了解unity以及简单游戏的流程，不过本书有点老了，17年的untiy相对于现在的unity已经有比较大的改变，不过基本逻辑并没什么变化，主要是基于代码的一些弃用和更新。如果有兴趣的话，19，20，21版本的unity都可以很好的适应我这次的代码。我是用的是unity2020.3，如果你不知道下载那个，用推荐的稳定版就好了。

创建项目

对Unity基本布局的认识

虽然unity已经有中文版了，不过我还是推荐用英文版的，这样在编写代码的时候可以校对参数。对于布局认知，一开始都是眼花缭乱的，尤其对于初学者，不管是建模软件还是unity这种软件，还有一些工具软件，全英的布局很容易成为劝退初学者的第一个要素。不过，对于初学者来说，一开始做的游戏也是很简单的，不用用到怎么多操作，所以逐步认识，在后续的练习可以巩固前面认识的操作，从而变得熟练。

对《拾苹果》原型分析

磨刀不误砍柴工，在做之前要分析好游戏内的各个组成成分（不用太详细），理解游戏以及其工作原理，以便于顺利地进行工程。

  • 游戏机制：用篮筐去接苹果树掉落的苹果。

游戏机制是很简单的，所以十分适合入门。从游戏机制中，可以分析到有三个物体——篮筐、苹果树和苹果。

篮筐：篮筐是由玩家控制来接苹果，接到可以得分，没接到可以判定一次“死亡”，我们在这里设置三个篮筐（可以根据自己的喜好设置篮筐个数），没接到一次苹果就会少一个篮筐，全部没接到便会结束游戏。

苹果：苹果从苹果树落下，垂直向下坠落。被篮筐接住，消失并得分。没接住，屏幕里的苹果全部消失，并减少一个篮筐，然后苹果树又重新开始掉落苹果。

苹果树：左右移动，掉落苹果，掉落苹果的时间是固定的（如果想要的话，可以让掉落的时间随机），苹果树会随机向左向右移动。

  • 分析物体动作

篮筐的动作：随着鼠标移动，接苹果

苹果的动作：下落，碰到地面让篮筐减少一个并让屏幕里的苹果全部消失

苹果树的动作：左右随机移动，掉落苹果

速写，我是一直在寻找有效练习，可以说我有一半的画作是有效练习的就不错了。看到朋友圈的同学发的画的比我好未免心生嫉妒，同时也会去查找原因。因为有些人本身就是在工设读的，我作为一个理工科学生，毕竟在绘画方面学的相比他们要少很多。可能我在我们班的绘画水平中是中间水平吧，但是在这个领域不比差，只比谁更强，所以有时候会emo😫。

作业既然没什么说的，做之前还会觉得很困难，可以说很多，做完后才发现只是个小事（建模再也不可能用alias了，太辣鸡了）。接下来说说我的2021吧。

2021刚开始就是军训+放假，可以说我的2021是在下学期开学之后才算开始。课业也多了很多，从第一学期的三门考试变到了七门考试（在电信的时候），学的那叫一个痛苦。不过翘课频率也增加了，主要还是翘数据结构，毕竟对我来说很简单。我花了2周时间对着视频教程做了小狐狸，当时生活巨不节律，喝奶茶健身加熬夜早起写代码，翘了很多课，搞得内分泌失调，长了好多痘痘😭（直到现在还有些没有消去），不过算是对unity有了些基本认知。

同时，多亏与wtw一次聊天，想去查一下转计算机学院的要求，突然发现已经开始转交互专业的time了，根据我查的资料，发现是早了一整个月的，才发现前两年是有疫情所以是线上的，差点错过（错过了宣讲会，不过根据现在的同学说，当时只有三个人去，七个老师给三个学生宣讲，🤣），于是准备了一个月，各种补知识，在排名比较中间的地方转专业成功了。

当时同时考7科，天天喝咖啡从早复习到晚上，就怕挂科。不过最终还是没挂科，最难的电路67险过（没想到在班里还可以排十几名，班里四十多个人😂），成功地转到了设计学院。

暑假稍微对做网站有点兴趣，于是花了一周时间简单学了三件套，加上不断地练习，练习到开学后十几天，搭好网站后就变成懒狗了（bushi），但是应付下学期的课程绰绰有余了。

到了中期，大作业不断接踵而至，有段时间每天2点睡觉8点起床去上课，那段时间真的是要累死，emo率也很高。学习了素描与色彩，对于绘画的初步认知不再是大一时的玩玩而已，也对下一步的绘画道路进行思考。

我的2021，简单来说，便是重新认识自我，确认目标，可能现在还不是特别清楚，因为时间不多了，我又有两条道路可以选择（游戏与设计），所以还不能坚定地立下来。

总的来说，还是学到很多，但是还不够，2022，唔，由不得不卷的理由了。

2021 广州核聚变

发表于 2021-11-28 更新于 2022-08-06 分类于 杂谈
本文字数： 2.7k 阅读时长 ≈ 2 分钟

认识机核

说起核聚变，就不得不说是如何知道机核的。如果不知道机核的话，基本上是不知道有这个活动的。当时明日方舟二周年活动——覆潮之下，特别克苏鲁，当时就打算了解一下克苏鲁，PV评论下有介绍故事背景，便搜索了印斯茅斯的阴霾，便接触了b站的机核网。到了暑假，实在是无聊，便打算听听电台，于是便打开了机核网页版，开始听各种各样的电台，听一堆大叔胡逼吹牛讲故事还挺有意思。

去核聚变之前的准备

准备在机核听电台的时候看到新的一次核聚变要来广州举办，想着我也没去参观过游戏展，于是便打算去了。那段时间ddl还特别多，11.20、11.21我两天都想去，于是我把ddl提前打算提前做完，那3周是真的一直肝作业，早上下午上课，晚上肝作业。约上2个朋友陪我参加第一天的核聚变，第二天打算自己一个人去玩，认为体验会有所不一样，事实也是，确实是个有意思的经历。

不幸的是，我在决定好了之后我们院的院运会安排在11.21，撞了日期很是糟糕，而且还必须参加开幕式🤕，班级选的入场表演我觉得不太好，于是参加彩旗队，一开始以为彩旗队会像大一原来的电信学院一样轻松，就开幕式举着旗子走过去，没想到现在这个学院要做好多动作，大概排练了六七次，每次半个钟😂。不过还是没参加任何一个项目，决定开幕式结束就准备溜。

核聚变第一天

从大学城南到达琶洲，只用坐3个地铁站，二十分钟左右就到了，还是蛮近的。当天有三个展览，一个广交会，一个车展，一个核聚变，我像个铸币一样跟着人流到了核聚变展馆的对面。到门口的时候发现不对劲，赶紧穿过地铁站到达保利世贸博览馆，还记得去年国庆去漫展也是到的这里。

等候同学，没想到他忘带身份证了，还好支付宝有电子身份证，虽然也没用到，最后也只是登记就过了😂。没想到的是广州公安借此冲业绩（ 下载国家反诈APP），虽然学校已经强制要求我们下载了，但是参加这个展览主要还是社会人士，于是因此排了很久。身份证验证的时候我反而比我忘带身份证的同学慢，人脸识别的机器对我来说太矮了，我一开始蹲下一直没检测到，看着其他列的别人先过去，还是十分尴尬的。

扫码正式进入场馆，领了一个蓝色day01手环，一个taptap集印章的手册，和一个核聚变展览手册。

是跟两个同学一起来，但是还有一个人没有到，于是我们打算大致逛一下。看了一会发现有很多好玩的，于是打算等那个同学来了之后再一起玩。于是到了独立游戏区，在去核聚变之前，我有在看机核发布的独立游戏展示的直播录像。在众多独立游戏之中，我决定先看看双相，是一个不错的平台跳跃游戏，游戏难度适中，解谜也挺不错的。玩到第七关卡在那了（总共有八关），此时已经十一点多了，另一个同学也来了，让他带了份麦当劳。我们准备在中午休息时吃完然后开始玩。

从12点多开始，我们现在taptap区游荡，一开始玩透视和深沉之火，都是非常有意思的游戏。不过深沉之火太多人玩了，根本排不到队伍，但是有缘跟开发者聊关于游戏开发的一些边边角角。为了集20个章，去得一个taptap的画册（事后有点后悔，感觉质量一般，不过毕竟不是买的，也不是那种很贵的画册）。除了前面的游戏，其中比较有意思的游戏，霓虹深渊，像素类横板跳跃rougelike弹幕游戏，运气比较欧，打三个boss，第一个打完就拿到了最好的激光枪，在那站撸，2分钟就完成了任务，获得了个马克杯，还算不错，接着断断续续的玩到3点多，终于集齐二十个章，便跑去核聚变的主场馆玩游戏。

首先是剑玉，一个挺好玩的玩具，还可以集印章，每个人有三次挑战机会，我在第三次的时候挑战成功了。

接着到旁边去玩了鬼灭之刃的火神雪风谭，可以说就是3d的拳皇，十分有意思，（第二天我也去玩了，也就是这个游戏，让我有了一个奇妙的经历），同样也没有拍照留念😇。双人成行和地平线5排队的人太多了，于是到任天堂大乱斗区玩，2v2，任天堂的游戏就是好玩。此时已经四点了，去看看主舞台的最后活动。

今日核聚变结束，晚到的同学不在广州读大学，这次来是为了顺便找女朋友，于是剩下的两个单身狗到旁边的商城吃饭，港式茶餐厅，甜腻的要死，有个鸡脆骨都没吃多少。写的时候才发现自己并没有拍照(;´༎ຶД༎ຶ`)，虽然过于甜腻，但是卖相还是不错的（大众点评找到的图。 报菜名：滑蛋牛肉饭（看着分量少，一开始以为不够吃，没想到糖加了好多，太腻歪了），鸡脆骨（好吃，但是最后吃不下了），小食拼盘（香肠、猪肠粉、鱼蛋、鸡翼尖），西多士（无敌好吃😋，12块钱9个，全场最值的）

晚上回到宿舍已经是七点多了，收拾下东西洗个澡发现自己忘记做限时答题了，好在周日还有得答题，定下闹钟。

核聚变第二天

周日院运会开完已经是10点半了比官方手册预计的时间还延迟了半个钟（😂100米 10点开跑），在站的时候还收到消息说我的大作业建模的模型不能用于3d打印，只好找淘宝修复并打印，花了150大洋😭😭。一结束，在小卖部买了两个面包来应付中午吃的东西，坐上地铁11点10分就进入场馆了（前面并没有排队，估计是太晚了）。

一开始先想着把章给收集了，然后再玩个痛快，先去玩了暖雪，挺容易上手的。然后专门排了40分钟玩地平线5，手柄+大屏幕真是爽爆了，不过没去排有方向盘的队伍😢（很可惜，而且我还没挑战成功，第一次用手柄玩赛车游戏）。

然后去玩帝国时代IV，还是不错的游戏，以前挺喜欢玩的游戏类型，现在更喜欢动作类和解谜类游戏了。又排了一下队玩完异星求生，决定再玩一个游戏，鬼灭之刃，这个是我认识那位陌生人的契机。原本只打了一把5局三胜的没有聊起来，但是看到工作人员没有过来，我们便再开了一把，于是便开始聊起来了，谈论自己平常玩什么游戏。然后带他去集章，他是周六上班，周日过来的（只有单休，打工人命苦啊），从北方一个人过来来广州一个外贸公司当设计(特别有勇气，敬佩)。带他去玩双人冰壶，任天堂，还有霓虹深渊，可惜的是霓虹深渊今天翻车了，没能成功再次拿到礼品🤣。

在闭幕前终于玩到深沉之火了，玩完后我们便离开场馆，我成功突破社恐的第一步，邀请他一起吃饭，仍然是到旁边的六元素吃饭，不过这次是大师兄（陕西面）。又没拍照🤐（没有拍吃的饭的习惯）。他请了我一个肉夹馍，原本想要全请我的，但是毕竟他是第一年上班，他全请还是不太好。

在地铁站分道扬镳，他与我是相反的路线，他住白云区，我住番禺区。

结束后

抽奖抽了两个c奖（帆布袋和桌布），虽然很眼馋S奖（Xbox series x），但是自己是非酋。

第一次写那么长的博客，有点像流水账了（其实就是了），赶大作业去了——润~
